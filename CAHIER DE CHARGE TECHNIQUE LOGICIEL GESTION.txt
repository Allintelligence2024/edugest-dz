## 1.2 Contexte & Justification

```
PROBLÈME IDENTIFIÉ :
┌─────────────────────────────────────────────────────────┐
│ Les centres de cours particuliers algériens gèrent      │
│ encore manuellement :                                   │
│  ❌ Les cahiers d'inscription papier                    │
│  ❌ Les paiements en espèces non tracés                 │
│  ❌ Les emplois du temps sur Excel                      │
│  ❌ Les relances des parents par appel téléphonique     │
│  ❌ Les bulletins imprimés à la main                    │
│  ❌ La paie des enseignants calculée manuellement       │
└─────────────────────────────────────────────────────────┘

SOLUTION PROPOSÉE :
┌─────────────────────────────────────────────────────────┐
│ Une plateforme SaaS centralisée, accessible sur web     │
│ et mobile, gérant l'intégralité des opérations d'un    │
│ centre de cours particuliers ou école privée, adaptée   │
│ au contexte algérien (paiement local, fiscalité,        │
│ calendrier national, curricula)                         │
└─────────────────────────────────────────────────────────┘
```

## MODULE M02 — PLANNING & EMPLOI DU TEMPS

### Règles de gestion

```
RG-PLAN-01 : Un enseignant ne peut avoir 2 cours au même créneau
RG-PLAN-02 : Une salle ne peut accueillir 2 groupes simultanément
RG-PLAN-03 : Créneaux disponibles : 7h00 → 22h00 (par défaut)
RG-PLAN-04 : Durée minimale d'un cours : 1h00
RG-PLAN-05 : Durée maximale d'un cours : 3h00
RG-PLAN-06 : Pause minimale entre 2 cours (même enseignant) : 15 min
RG-PLAN-07 : Respect du calendrier scolaire algérien (vacances)
RG-PLAN-08 : Alerte automatique en cas de conflit de planning
```

### Structure des créneaux


# PARTIE 6 — SÉCURITÉ & CONFORMITÉ

---

## 6.1 Sécurité des Données

```
MESURES DE SÉCURITÉ :

🔐 AUTHENTIFICATION
   ├── JWT Token (expiration 1h)
   ├── Refresh Token (7 jours)
   ├── 2FA par SMS (optionnel)
   └── Blocage après 5 tentatives échouées

🛡️ CHIFFREMENT
   ├── HTTPS/TLS 1.3 obligatoire
   ├── Données sensibles : AES-256
   ├── Mots de passe : bcrypt (cost 12)
   └── Backup chiffré

🔒 CONTRÔLE D'ACCÈS (RBAC)
   ├── Rôles définis par tenant
   ├── Permissions granulaires
   └── Log de toutes les actions

🔍 AUDIT & LOGS
   ├── Journal des connexions
   ├── Historique des modifications
   └── Alertes d'activités suspectes
```

## 6.2 Conformité Légale Algérienne

```
LOI 18-07 du 10 juin 2018 (Protection des données personnelles) :
  ✅ Consentement explicite des utilisateurs
  ✅ Droit d'accès, modification, suppression
  ✅ Hébergement en Europe (compatible)
  ✅ DPO désigné (Délégué à la Protection des Données)
  ✅ Mention légale et politique de confidentialité

FISCALITÉ ALGÉRIENNE :
  ✅ Conformité IRG (barème 2026)
  ✅ Conformité CNAS / CASNOS
  ✅ TVA (si applicable)
  ✅ Numéro NIF/NIS sur les factures
```

```sql
-- ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

-- Isolation multi-tenant (sur toutes les tables)
CREATE INDEX idx_eleves_tenant       ON eleves(tenant_id, statut);
CREATE INDEX idx_enseignants_tenant  ON enseignants(tenant_id, statut);
CREATE INDEX idx_cours_tenant        ON cours(tenant_id);
CREATE INDEX idx_paiements_tenant    ON paiements(tenant_id, date_paiement);

-- Recherche rapide
CREATE INDEX idx_eleves_search       ON eleves(tenant_id, nom, prenom);
CREATE INDEX idx_eleves_niveau       ON eleves(tenant_id, niveau_scolaire);
CREATE INDEX idx_notes_eleve         ON notes(eleve_id, evaluation_id);
CREATE INDEX idx_presences_seance    ON presences(seance_id, eleve_id);
CREATE INDEX idx_factures_statut     ON factures(tenant_id, statut, date_echeance);
CREATE INDEX idx_seances_date        ON seances(cours_id, date_seance);

-- Audit
CREATE INDEX idx_audit_tenant_date   ON audit_logs(tenant_id, created_at DESC);
```

```
HTTP 404 — NOT_FOUND             (Ressource introuvable)
HTTP 409 — CONFLICT              (Conflit planning, doublon)
HTTP 422 — VALIDATION_ERROR      (Données invalides)
HTTP 429 — TOO_MANY_REQUESTS     (Rate limit dépassé)
HTTP 500 — SERVER_ERROR          (Erreur interne)

E001 — PLANNING_CONFLICT         (Conflit emploi du temps)
E002 — SALLE_FULL                (Salle déjà occupée)
E003 — GROUPE_COMPLET            (Groupe au maximum)
E004 — PAIEMENT_DEJA_EFFECTUE    (Doublon paiement)
E005 — ELEVE_DEJA_INSCRIT        (Déjà dans ce groupe)
E006 — SOLDE_INSUFFISANT         (Paiement en ligne)
E007 — TENANT_EXPIRED            (Abonnement expiré)
E008 — QUOTA_DEPASSE             (Limite plan atteinte)
```

```yaml
version: '3.8'

services:
```

## 4.5 GitHub Actions — CI/CD Pipeline

```yaml
# .github/workflows/deploy.yml
name: 🚀 EduGest DZ — CI/CD Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  # ━━━━━━━━━━━━━━━━━━━━━━━━
  test-backend:
    name: ✅ Tests Backend Laravel
    runs-on: ubuntu-latest
    
    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_DB: edugestdz_test
          POSTGRES_USER: test_user
          POSTGRES_PASSWORD: test_pass
        ports: ['5432:5432']
      
      redis:
        image: redis:7
        ports: ['6379:6379']
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup PHP 8.3
        uses: shivammathur/setup-php@v2
        with:
          php-version: '8.3'
          extensions: pgsql, redis
      
      - name: Install Dependencies
        run: composer install --no-interaction
      
      - name: Setup Environment
        run: |
          cp .env.testing .env
          php artisan key:generate
          php artisan migrate --seed
      
      - name: Run Tests
        run: php artisan test --coverage

  # ━━━━━━━━━━━━━━━━━━━━━━━━
  test-frontend:
    name: ✅ Tests Frontend React
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      
      - run: npm ci
      - run: npm run lint
      - run: npm run test
      - run: npm run build

  # ━━━━━━━━━━━━━━━━━━━━━━━━
  deploy-staging:
    name: 🌐 Deploy → Staging
    needs: [test-backend, test-frontend]
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/develop'
    
    steps:
      - name: Deploy to Staging
        run: |
          echo "Déploiement sur staging.edugestdz.com"
          # SSH + rsync + artisan migrate

  # ━━━━━━━━━━━━━━━━━━━━━━━━
  deploy-production:
    name: 🚀 Deploy → Production
    needs: [test-backend, test-frontend]
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    
    steps:
      - name: Deploy to Production
        run: |
          echo "Déploiement sur api.edugestdz.com"
          # SSH + rsync + artisan migrate + config:cache
```
