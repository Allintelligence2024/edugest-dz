Prochaines priorités — Dans l'ordre d'impact
1
🔒 Branch protection sur main — MAINTENANT
GitHub → Settings → Branches → Add rule → "main" → Require PR + CI green avant merge.
Sans ça, un push direct sur main peut casser le projet en 1 commande. C'est 5 minutes de travail, coût zéro, risque infini si non fait.
⏱ 5 minutes · Manuellement sur GitHub · Aucun code
2
🔗 Intégrations facturation — Transport + Cantine → Facture élève
M09 (transport) et M10 (cantine) existent dans le code mais leurs tarifs ne sont pas encore ajoutés à la facture mensuelle élève de M03. Sans ça, les directeurs doivent facturer manuellement — ça tue la proposition de valeur. Priorité : ajouter des lignes de facture automatiques quand un élève est inscrit au transport ou à la cantine.
⏱ 3–4 jours · Modifier FacturationService + tests
3
📦 M11 — Stock & Inventaire mobilier
Le seul module opérationnel encore absent. Les directeurs algériens ont une obligation d'inventaire annuel. Chaises, tables, tableaux, équipements — tout se perd sans traçabilité. Le stock cuisine de M10 existe déjà → M11 en est une extension naturelle pour le mobilier et les fournitures pédagogiques.
⏱ 1–2 semaines · Migration + CRUD + QR code + alertes
4
🔧 M14 — Entretien Bâtiment
Tickets de panne, prestataires, entretien préventif. Module simple (CRUD + alertes) mais fortement demandé terrain. Chaque réparation doit être tracée et liée aux dépenses M13. C'est la dernière brique de l'ERP scolaire complet.
⏱ 1 semaine · Migration + CRUD + lien M13
5
📱 Application Mobile — Compléter les screens manquants
L'app mobile (React Native Expo) existe mais couvre seulement le rôle parent. Les screens enseignant (planning, présences, notes) et admin (tableau de bord) manquent. C'est bloquant pour une démo terrain convaincante — un directeur voudra voir l'app mobile fonctionner sur son téléphone.
⏱ 3–4 semaines · React Native screens + intégration API
6
💳 Paiement CIB/Dahabia — Activer Satim
Le code est écrit (30% fait). Ce qui bloque : le contrat commercial avec Satim et l'accès au sandbox. Démarche administrative, pas technique. À lancer en parallèle des autres priorités — les délais administratifs sont longs.
⏱ Côté code : 1 semaine · Côté admin Satim : 4–8 semaines (démarche externe)
7
🎯 1 Centre Pilote en production — LA vraie priorité absolue
Le code est prêt pour une démo complète. Inscriptions, planning, notes, absences, bulletins, personnel, transport, cantine, budget — tout fonctionne. Aucune feature supplémentaire ne remplace un directeur qui utilise l'app lundi matin. C'est le seul signal qui compte pour valider que le produit a de la valeur.
⏱ Cette semaine · Aller physiquement dans 10 centres · 0 ligne de code
